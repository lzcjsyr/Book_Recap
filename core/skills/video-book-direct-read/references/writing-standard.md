# Video Book Script Writing Standard

把源资料转成适合抖音和微信视频号的中文音频解说稿。目标不是复述资料，也不是读书报告，而是替中国观众做几分钟内的「第一遍速读」：讲清最值得知道的是什么、为什么重要、和自己有什么关系，以及为什么值得继续读或继续聊。

## Workflow

不要一轮直接输出。按「理解 -> 定角度 -> 写初稿 -> 多轮修订 -> 包装输出」执行：

1. 资料理解：只依据已经覆盖或有证据支撑的资料写作。不虚构、不偷换论点、不强加资料没有表达的立场。
2. 类型判断：静默判断资料类型：经济/制度、历史、认知/心理、社会理论/经典、当代报告/AI/公共议题。
3. 角度设计：提取核心问题、作者真正强调的重点、最有价值的解释框架，以及一个跟观众有关的锋利角度。
4. 初稿写作：按理解和留存重组内容，不按目录平推。把最值得听的内容放到前面。
5. 多轮修订：按 `revision-workflow.md` 分别保存忠实性稿、结构稿、口语稿和终稿，再基于相邻版本差异写修订审计，至少完成忠实性、结构节奏、传播口播、格式四轮修订。
6. 包装输出：修订完成后，再从终稿生成标题、封面文案、金句、评论/转发引导，并只输出严格 JSON。

## 最高优先级

- 忠于资料：不虚构、不偷换、不强加原资料未表达的立场。
- 先讲清「是什么、为什么重要」，再追求金句和传播感。
- 找到一个叙事的主线。不机械总结，不按章节或目录平推。
- 口吻要像一个读透资料、也懂普通人处境的人在讲。
- 让观众觉得「这事和我有关」：有冲突、反常识、社交货币和讨论空间。
- 有人文关怀。

## 开头

- 前 100 字内不要寒暄、空话或背景铺垫。
- 开篇第一句话必须提出一个牵引全文的问题；问题必须来自原资料的真实矛盾或现实困惑，不要为了互动硬造。
- 引出反常识的冲突。要有收益承诺，让观众知道听完能获得什么。
- 自然引出作品名 `《XX》`。
- 不要先讲作者生平，除非这对理解问题本身不可缺少。

## 结构和节奏

- 围绕理解、留存和共鸣重组资料。
- 最值得听的内容放前面，不平均分配篇幅。
- 背景只为解释因果、结构、选择、约束、代价或后果服务。
- 约每 250 字要有一次认知推进：更尖锐的问题、反常识判断、案例/场景、现实对照、反转或代价。
- 避免连续两段只讲背景或概念。

## 传播感

- 重要观点要落到具体的人、事、场景、制度、职业、生活选择、风险、机会或取舍。
- 至少写出两处「被说中」的感觉，让观众觉得自己经历过但没说清。
- 自然埋入三个评论触发点，来自现实分歧、经验对照、代价判断或价值张力。
- 至少有一句观点鲜明、好记、有记忆点的金句。
- 不制造廉价对立，不为了互动硬造问题。

## 文风

- 使用简体中文。
- 口语化、大白话，适合直接口播。
- 避免翻译腔、学术腔、营销腔、读书报告腔、课堂讲义腔、AI 总结腔。
- 每句尽量只有一个核心意思，避免抽象名词堆叠。
- 信息密度要高，但不要说教，也不要煽情过度。

## 内容类型适配

- 经济学 / 政治经济 / 制度分析：重点讲利益结构、激励机制、制度约束、代价与后果，解释钱、权、信用、政策、市场和普通人处境之间的关系。
- 历史 / 历史解释 / 历史人物与结构：重点讲时代背景、关键选择、资源分配、制度演变和历史转折背后的真实因果。
- 认知科学 / 心理学 / 行为研究：重点讲结论、实验、机制、常见误区和适用边界，区分扎实研究、解释框架和仍有争议的部分。
- 社会理论 / 思想经典 / 社会科学名著：重点讲作者解释的社会现象、挑战的流行理解、提出的关键框架，以及它如何照到现实问题。
- 当代报告 / 机构研究 / AI 社会影响与公共议题：区分证据判断、推演、倡议和价值立场，优先落到就业、教育、治理、创新、安全和日常生活。

## 避免

不要使用这些表达或明显变体：

- "本书讲述了……"
- "作者认为……"
- "本书分为三部分……"
- "接下来我们来看……"
- "总的来说……"
- "引出了这本书里……"
- "评论区告诉我"
- "你怎么看"
- "学到了扣1"
- "转发给有需要的人"

## 结尾

结尾要重新钉住作品的核心价值，给观众一个现实中的思考方向、行动提醒或认知坐标，并自然留下一个适合继续讨论的判断或争议点。结尾要简洁、有余味，不要像课程收尾。

## 开场金句

- 金句不是标题，也不是总结句，要像开场前能让人停一下的判断。
- 优先写成反常识判断、代价判断、现实刺痛或价值冲突。
- 不要写空泛鸡汤、营销口号、万能句。
- 每条 15-30 字，能单独成立，也能自然引出口播开头。

## 修订流程

写完初稿后，不要直接进入最终 JSON。按 `revision-workflow.md` 把修订过程逐轮落盘，至少保存 `_draft_v1.txt`、`_draft_v2_faithfulness.txt`、`_draft_v3_structure.txt`、`_draft_v4_oral_style.txt`、`_draft_final.txt`、`_revision_audit.json`：

1. 忠实性修订：删掉无法从资料中支撑的判断，纠正偷换概念、过度推断和强行拔高。
2. 结构节奏修订：检查开头 100 字、主线、信息顺序、认知推进和背景比例。
3. 传播口播修订：把抽象句改成具体场景，把书面句改成口播句，补足金句、评论点和「被说中」的表达。
4. 格式修订：按本文“最终 JSON 输出契约”检查字段、数量、长度、JSON 可解析性，以及 JSON `content` 是否是不分段终稿。

修订审计必须基于每一组相邻版本的真实差异来写，写清每轮目的、输入文件、输出文件、具体原句和新句。某一轮没有实质差异就不能声称完成该轮，必须继续改稿。不要只写“已优化”“已检查”“符合要求”。

## 最终 JSON 输出契约

最终只输出严格 JSON，不要使用 Markdown，不要添加额外解释。若运行环境要求保存文件，保存到用户指定 raw JSON 路径，并确保 Python `json.loads` 可解析。

```json
{
  "source_name": "作品本身标题，不用文件名后缀",
  "video_titles": ["3条发布标题候选，互不重复，信息完整"],
  "cover_titles": ["3条封面主标题，4到10字，短、强、易扫读"],
  "cover_subtitles": ["3条封面副标题，15字以内"],
  "golden_quotes": ["3条开场金句，观点鲜明，像一句能让人停下来的真话"],
  "comment_hook_options": ["3条评论引导，优先判断句、对照句、追问句，不像运营话术"],
  "share_hook_options": ["3条转发引导，像观众看完后会自然转发的话"],
  "content": "完整口播终稿，约3500字，不分段，不写Markdown或小标题",
  "total_length": 0,
  "target_segments": 0
}
```

- `content` 由 `_draft_final.txt` 去掉换行和多余空白生成，不改正文措辞。
- `total_length` 写实际 `content` 字符数。
- `target_segments` 使用用户或程序传入的目标段数，不要照抄示例值。
- JSON 没有尾随逗号。
- 所有字符串正确转义。
- 每个候选数组各 3 条，互不重复。
- `content` 没有分段、小标题或列表格式。
